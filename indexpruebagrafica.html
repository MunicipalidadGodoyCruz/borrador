Pie

es un plugin de Leaflet que simplifica la creación de mapas de símbolos proporcionales basados en gráficos de tarta.
La inspiración de esta librería es una visualización clásica de Charles Joseph Minard, Carte des quantités de viande (1858). Este tipo de representación es una herramienta útil para el análisis territorial porque es capaz de localizar y comparar valores absolutos y su desglose por subcategorías.

is a Leaflet plugin that simplifies the creation of proportional symbol maps based in pie charts.
Inspiration came from a classic data visualization of Charles Joseph Minard, Carte des quantités de viande (1858). This kind of representation can be a useful tool for territorial analysis because it can locate and compare absolute values and its break-down in categories.

Primeros pasos

Getting started

Pie usa una clase base L.Pie.Dataset. Esta es el contenedor de los datos y de sus modos de representación, contenidos en la clase L.Pie.Display. Un display es un conjunto de instrucciones de representación de datos —como los colores, etiquetas y tamaños—. Así­que para empezar tienes que hacer tres cosas:
  1. Cargar los archivos del plugin —LeafletPie.min.js y LeafletPie.min.css.
  2. Instanciar un dataset pasando un array con los datos que quieres representar.
  3. Añadir al menos un display para representar esos datos (y activarlo).
Pie uses a base class named L.Pie.Dataset. This one is the container of the data to be rendered and the ways of representing it, encapsulated in the class L.Pie.Display. A display is a set of instructions to represent data—like colors, labels, sizes...—. So, to start you've to do three things:
  1. Load plugin files as usual—LeafletPie.min.js and LeafletPie.min.css.
  2. Instantiate a dataset, passing to it an array of the elements that compose the dataset.
  3. At least add a display to the dataset and activate it.
$.getJSON('data.json', function(data){
    // Instantiation expects a JSON array or dictionary composed
    // by objects like this one:
    // { "name"  : "Portugal",
    //   "coords": [38.74, -9.19],
    //   "data"  : [1097.85, 1215.68, 212.30] }
    // Only coords and data are mandatory
    var dataset = L.Pie.dataset(data);
    // Add a named display
    dataset.addDisplay('display_label', {
        min_size : 20,
        compensation : false,
        categories : [
            { label: 'Category A', color: '#080A09' },
            { label: 'Category B', color: '#354B58' },
            { label: 'Category C', color: '#C93529' },
        ],
    });
    dataset.displaySet('display_label', map);
}
El marcado generado para cada gráfica es un SVG, donde cada 'porción de la tarta' es un path. A cada path se le añade una clase que es el resultado de slugificar su etiqueta para facilitar operaciones JS/CSS. Por ejemplo una categoría con etiqueta «Número de socios» añadirá­al path una clase cat--numero-de-socios. Los valores necesarios se añaden como atributos data al elemento para poder acceder a ellos mediante javascript, pudiendo accediendo al diccionario dataset del nodo. (Ver ejemplo Hover callbacks más abajo.) Las opciones min_size y compensation determinan los tamaños de estos elementos. La primera es el diámetro del elemento más pequeño del dataset, y la segunda establece si queremos aplicar el método de compensación de Flannery a la hora de calcular los tamaños relativos de los elementos. Este método hará los iconos más grandes pero compensará la comparación de tamaños relativos. Artículo sobre los métodos cognitivos en el diseño de mapas, con la fórmula de Flannery empleada aquí
The generated markup for each pie chart will be a DivIcon, with a SVG inside, wherein each 'slice' is rendered as a path. A class will be added to each of these 'slices', result of slugifying its data label, to ease JS/CSS operations. For instance an item labelled as 'number of members' will have a class 'cat--number-of-members' appended to it. Color, label and value will be appended as data attributes as well, allowing to retrieve them using javascript (see Hover callback example below). min_size and compensation options determine the size of pies. The first one will set the diameter of the smallest data item in the set and the second one will activate Flannery's compensation when calculing the rest of sizes. Compensation will yield bigger items but 'compensated', that's it, it will take into account the way in which our brain perceives circular areas, making comparations between elements more accurate. Article about cognitive map-design, including the Flannery's formula used here.

Tooltips

Puedes añadir tooltips como opción de configuración del display. Para ello tienes que pasar una opción default-hover al display con valor true. Puedes añadir clases CSS al contenedor del tooltip con la opción hover_classes. Las cantidades mostradas en el tooltip se sufijarán con la unidad que indiques en el parámetro units. La librería dispone de varias clases genéricas para dar estilo al tooltip, dentro del namespace lpie. En este ejemplo se usan lpie-light y lpie-shadow, para usar un tema base claro y poner una sombra al tooltip. Actualmente los tooltips se resuelven exclusivamente con CSS, de manera que pueden quedar parcialmente ocultos bajo otros marcadores.

You can configure the display to show tooltips by default when hovering a pie, passing to it a default-hover option set to true. To customise it you can add HTML classes to the tooltip container, passed as a String value to hover_classes display's option (for example "class-a class-b class-c"). The tooltip will show a complete breakdown of pie's values, suffixed by the unit passed in the argument units. The plugin has a few generic classes to set default styles, prefixed by the namespace 'lpie', as 'lpie-light', lpie-dark' and 'lpie-shadow'.
Disclaimer: To keep things simple, tooltip effect is solved with CSS right now, so it can be partially hidden under leaflet markers. 

dataset.addDisplay('display_label', {
    // ...
    categories : [
        { label: 'Category A', color: '#080A09' },
        { label: 'Category B', color: '#315041' },
        { label: 'Category C', color: '#C93529' },
    ],
    units          : "€",
    default_hover  : true,
    hover_classes  : 'lpie-light lpie-shadow'
});
dataset.displaySet('display_label', map);

El marcado del tooltip sigue la estructura:

Tooltip's markup follows this structure: 

<ul class='lpie-icon__tooltip {{ hover_classes }}'>
     <li class='lpie-icon__tooltip-item__legend-item'>
        <p class='lpie-icon__tooltip-label'>
           <i class='lpie-icon__tooltip-color-icon'></i>
           {{ label }}
        </p>
        <p class='lpie-icon__tooltip-value'>
           {{ value }} {{ unit }}
        </p>
     </li>
    // ...
</ul>

Leyenda

Legend

Otra necesidad común es la de generar un cuadro explicativo para el dataset. Siguiendo un principio DRY la librería dispone de un control de Leaflet para generarlo automáticamente a partir de un display. Al instanciar la leyenda se le ha de pasar este, que usará para crear automáticamente la lista de categorías y colores asociados. Del mismo modo que con los tooltips puedes añadir clases CSS al contenedor en el valor de la opción legend_classes.

To have an explicative legend is a common need for this kind of displays. Following a DRY convention, the plugin has a Leaflet control to render it automatically from a given display and a set of arguments (title, description, source). The display will be used to fetch label and color per each category and append a category list to the legend. As with tooltips you can pass a legend_classes argument to customise the look of the legend through CSS. 

// Add optionally a named legend
dataset.addLegend('legend_label', {
    title         : 'Lorem ipsum',
    description   : 'Velit nisi deserunt...',
    source_url     : 'https://lipsum.com/',
    source_text    : 'https://lipsum.com/',
    position       : 'topright',
    legend_classes : 'lpie-light'
}, 'display_label');
dataset.displayLegend('legend_label', map);

El marcado de la leyenda sigue la estructura:

Legend's markup follow this structure: 

<div class="lpie-legend leaflet-control">
    <p class="lpie-legend__title">
        {{ title }}
    </p>
    <p class="lpie-legend__description">
        {{ description }}
    </p>
    <ul class="lpie-legend__categories">
        <li class="lpie-legend__categories-item">
            <span class="lpie-legend__color-icon" style="color: {{ color }}"></span>
            <span class="lpie-legend__categories-label">{{ label }}</span>
        </li>
        // ...
    </ul>
    <div class="lpie-legend__source">
        <span class="sl_cl">Source: </span>
        <a href="https://lipsum.com/" target="_blank">https://lipsum.com/</a>
    </div>
</div>

Popups

Puedes pasar al display el nombre de una función que será la que retorne el HTML del popup de leaflet. Esta función tendrá acceso —en el ejemplo la variable item— a campos del elemento correspondiente dentro del dataset.

Markers can have regular leaflet popups attached to them. To set the markup of the popup you can pass to the display the name of a function that will have access to item data in the dataset and that will return the markup to be rendered. 

function createPopup(item){
    return "<h3>" + item.name + "</h3><p>Lorem ipsum</p>" ;
};
// Add a named display
dataset.addDisplay('display_label', {
    // ...
    categories : [
        { label: 'Category A', color: '#080A09' },
        { label: 'Category B', color: '#315041' },
        { label: 'Category C', color: '#C93529' },
    ],
    // ...
    popup : createPopup,
});
dataset.displaySet('display_label', map);

Hover callbacks

Puedes pasarle una función al display para que se ejecute cuando la usuaria haga hover sobre los marcadores del mapa. Abre la consola para ver los resultados de este callback.

You can set a custom callback when the user hovers the pie to trigger custom events or to create different kinds of tooltips. To set a custom callback pass the name of the function to the display. Notice how in the previous case you passed the function whilst here you pass the function name 

function my_callback(item, parent){
    // Access using item.dataset to values, colors and categories
    // Parent is useful for DOM manipulation
    console.log(item.dataset) ;
};
// Add a named display
dataset.addDisplay('display_label', {
    // ...
    categories : [
        { label: 'Category A', color: '#080A09' },
        { label: 'Category B', color: '#315041' },
        { label: 'Category C', color: '#C93529' },
    ],
    // Pass the name as string, unlike popups
    hover_callback  : 'my_callback';
    // ...
});
dataset.displaySet('display_label', map);
